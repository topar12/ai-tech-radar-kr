제품 기준

완성형 제품의 북극성은 "읽을거리"가 아니라 "판단"이다. 모든 기능은 아래 질문에 답해야 한다.

오늘 무엇이 바뀌었나

모델 출시, 오픈소스 급상승, 논문, 기업 전략, 규제, 인프라 변화를 하나의 흐름으로 보여준다.

왜 중요한가

화제성, 기술성, 실무 적용성, 국내 관련성, 리스크를 분리해 해석한다.

지금 무엇을 해볼까

써볼 도구, 추적할 키워드, 읽을 원문, 도입 전 체크리스트를 행동 단위로 제안한다.

aitrends.kr 대비 방향

aitrends.kr는 한국어 AI 뉴스 피드, 트렌딩, 커뮤니티, 기술 태그, 용어사전을 이미 제공한다. 우리가 이기려면 더 예쁜 피드가 아니라 더 나은 판단 구조가 필요하다.

완성 상태

완성형 한 문장 글로벌 AI 신호를 한국 시장과 실무자의 의사결정 언어로 번역하는 AI 테크 레이더.

오늘의 AI 레이더

핵심 5개 변화, 급상승 기술, 기업별 움직임, 개발자용 업데이트, 관심 분야 알림을 첫 화면에 배치한다.

이슈 클러스터

같은 사건을 여러 출처와 함께 묶고, 원문별 관점 차이와 시간 흐름을 보여준다.

실사용 검증

GitHub 유지보수, 모델 라이선스, 추론 비용, 한국어 적합성, 보안/품질 리스크를 별도 점수로 표시한다.

사용 깊이 설계

모든 화면은 같은 원칙을 따른다. 처음에는 판단에 필요한 최소 정보만 보여주고, 사용자가 더 알고 싶을 때만 근거와 원문을 단계적으로 연다.

스냅샷

제목, 한 줄 결론, 중요도, 변화 방향, 카테고리만 노출한다. 바쁜 사용자는 여기서 멈춰도 된다.

판단 요약

무슨 일, 왜 중요, 누가 영향, 지금 할 일을 4블록으로 제공한다. 읽기가 아니라 판단을 돕는다.

검증 정보

출처 수, 공식 발표 여부, GitHub/HF/arXiv/HN 신호, 리스크를 보여준다. 신뢰를 사용자가 확인한다.

원문 탐색

타임라인, 출처별 관점 비교, 관련 이슈, 원문 링크로 이어진다. 깊게 보는 사람을 위한 층이다.

사용자 여정

같은 AI 이슈라도 사용자의 목적에 따라 앞에 보여줄 정보가 달라야 한다. 개인화는 필터가 아니라 우선순위와 표현 방식의 변화다.

AI 활용 실무자

내 업무에 바로 쓸 수 있는 변화인지 알고 싶다.

첫 10초영향도, 대상 직무, 실행 가능성 배지 확인
첫 3분왜 중요한지, 지금 해볼 행동, 유사 변화 비교 확인
깊게관련 툴/모델, 국내 적용 가능성, 추적 타임라인 확인

의사결정자

도입할지, 지켜볼지, 무시할지 판단하고 싶다.

첫 10초전략적 중요도, 시장 영향, 리스크 표시 확인
첫 3분기업 도입, 산업 영향, 투자/시장, 규제 탭 확인
깊게도입 판단 메모, 30일 변화 브리핑, 팀 공유용 요약 확인

학습자

무엇부터 이해하고 공부해야 하는지 알고 싶다.

첫 10초쉬운 해설, 핵심 개념, 배워야 할 키워드 확인
첫 3분초보자 해설 모드와 관련 학습 경로 확인
깊게기초 개념, 최신 변화, 실습 자료, 다음에 볼 것 연결

콘텐츠 제작자와 리서처

화제가 될 신호와 신뢰할 수 있는 맥락이 필요하다.

첫 10초급상승 주제, 논쟁 주제, 새 키워드 확인
첫 3분OpenAI, 에이전트, 로컬 LLM, 규제 등 주제 묶음 확인
깊게찬반 논점, 이전 흐름, 국내 맥락, 콘텐츠 구조 확인

화면별 UX 설계

제품의 각 화면은 설명문을 길게 붙이지 않는다. 반복되는 위치, 배지, 짧은 확장, 역할별 전환으로 사용자가 스스로 길을 찾게 한다.

홈

스크롤 전에 오늘의 AI 흐름을 판단하게 한다.

  • 상단 첫 줄: 오늘 AI 업계에서 실제로 바뀐 것 5개
  • 각 카드: 이슈명, 한 줄 결론, 중요도, 변화 방향, 영향 대상
  • 긴 설명 금지. `왜 중요`를 누르면 2줄만 확장
  • 내 Watchlist 변화는 홈의 보조 영역이 아니라 핵심 영역으로 배치

피드

기사 목록이 아니라 이슈 클러스터 목록으로 보이게 한다.

  • 정렬: 중요, 빠르게 확산, 실무 적용, 국내 영향, 최신
  • 출처 표시는 `공식 1, GitHub 2, 커뮤니티 4, 논문 1`처럼 압축
  • 필터 칩: Models, Agents, Infra, Open Source, Research, Business, Policy
  • 카드에서 바로 저장, 관심 등록, 숨기기, 원문 열기 가능

이슈 상세

읽는 페이지가 아니라 판단 페이지로 설계한다.

  • 최상단은 기사 제목이 아니라 결론 문장
  • 바로 아래 4블록: 무슨 일, 왜 중요, 누가 영향, 지금 할 일
  • 신뢰도 스트립: 공식 출처, 출처 수, 업데이트 시간, 연결 신호
  • 하단 탭: 타임라인, 출처 비교, 실사용 검증, 관련 이슈

트렌딩

뉴스가 되기 전 조기 신호를 잡는다.

  • 단순 랭킹보다 `왜 뜨는지`를 먼저 표시
  • 상승률, 발화 지점, 관련 클러스터, 실사용 가능성 표시
  • 오늘 갑자기 뜬 것과 3일째 유지되는 것을 구분
  • GitHub, HF, arXiv, HN 분리 보기와 통합 보기 모두 제공

브리핑

반복 방문의 이유를 만든다.

  • 타입: 아침 10분, 퇴근 전 놓친 이슈, 주간 회고
  • 각 항목은 한 화면에서 끝난다: 결론, 이유, 행동, 원문
  • 긴 근거는 접힌 상태로 시작
  • 끝에는 Watchlist에 추가할 키워드 3개만 추천

검색

결과 목록보다 답을 먼저 보여준다.

  • 검색 결과는 이슈, 기술/용어, 기업/제품, 원문으로 분리
  • 질문형 검색: `요즘 에이전트 프레임워크 뭐 봐야 해?`
  • 답변 뒤에 핵심 후보, 비교 기준, 최근 이슈, 읽을 원문 연결
  • 검색 결과에서 바로 `이 검색어 알림 받기` 제공

데이터 파이프라인

안정적인 1차 수집축은 RSS/Atom, GitHub, Hugging Face, arXiv, Hacker News다. Reddit, YouTube, SEC는 가치가 있지만 인증과 정책 관리가 있어 2차 레이어로 둔다.

1. Collect 공식 피드, API, 릴리즈, 논문, 커뮤니티 신호 수집
2. Normalize 출처별 데이터를 공통 Item 모델로 정규화
3. Cluster 같은 사건과 관련 신호를 이슈 단위로 묶기
4. Score 중요도, 속도, 실무성, 국내성, 리스크 점수화
5. Explain 한국어 구조화 해석과 역할별 요약 생성
6. Deliver 웹, 검색, 브리핑, 알림, Watchlist로 전달

기능 지도

Must

오늘의 레이더 홈 최신 피드 이슈 클러스터 한국어 구조화 요약 태그 체계 검색 모바일 최적화 원문 출처

Should

아침 10분 브리핑 주간 회고 레이더 맵 관심 태그 출처 관점 비교 저장 알림 역할별 개인화

Could

이슈별 토론 자연어 검색 AI 챗 탐색 팀 브리핑 API 제공 유료 리포트

검증 점수

GitHub 활동 라이선스 모델 크기 추론 비용 한국어 적합성 보안 리스크

신뢰, 개인화, 알림

AI 요약 제품에서 신뢰는 부가 기능이 아니라 제품의 기본값이다. 개인화와 알림도 더 많이 보여주기 위한 기능이 아니라 덜 읽고 더 잘 판단하게 만드는 장치여야 한다.

신뢰 UX

출처 유형을 `공식`, `논문`, `언론`, `커뮤니티`, `분석`으로 표시하고, 확실성 레벨을 `확정 발표`, `초기 보도`, `추정`, `논쟁 중`으로 분리한다.

개인화 UX

개인화는 필터 버블이 아니라 중요도 정렬 방식이다. `내게 중요`, `전체에서 중요`, `새롭게 뜨는 중`, `내가 놓친 변화`를 함께 보여준다.

알림 UX

알림은 새 글이 아니라 상태 변화 중심이다. `관심 레포 스타 급등`, `저장한 이슈에 공식 발표 추가`, `관심 모델 라이선스 논란`처럼 이유를 드러낸다.

Retention 루프

사용자는 새 뉴스 때문에 돌아오는 것이 아니라, 내 기준으로 중요한 변화가 업데이트되기 때문에 돌아온다. 저장, 숨김, 관심 없음, 더 보기 행동이 다음 방문의 레이더 품질을 개선해야 한다.

알림 기본값

기본은 주간 브리핑만 켠다. 실시간 알림은 사용자가 직접 켠다. 모든 알림에는 왜 받았는지와 바로 끄는 행동이 있어야 한다.

단계별 구현 계획

Phase 1

기반 제품

매일 자동으로 AI 핵심 이슈가 쌓이고, 웹에서 읽을 수 있는 상태를 만든다.

  • Next.js 웹 기본 구조
  • 기사/소스/태그/클러스터 DB
  • RSS, GitHub, HF, arXiv, HN 수집기
  • 한국어 구조화 요약
  • 오늘의 핵심 5개
  • 원문 링크와 출처 표시
Phase 2

aitrends.kr 추월 구간

뉴스 피드가 아니라 AI 업계 흐름을 이해하는 도구로 만든다.

  • 이슈 클러스터 상세
  • 출처별 관점 비교
  • 중요도 점수
  • 레이더 카테고리
  • GitHub/HF/arXiv 트렌딩 통합
  • 주간 회고 리포트
Phase 3

개인화

사용자가 매일 다시 들어올 이유를 만든다.

  • 관심 키워드/기업/레포/모델 팔로우
  • 개발자/PM/창업자/투자자/연구자 모드
  • 개인화 홈
  • 저장/나중에 읽기
  • 이메일/웹 푸시/Discord 알림
  • 놓친 이슈 요약
Phase 4

검증 레이어

읽는 서비스가 아니라 도입 판단 도구가 된다.

  • GitHub 유지보수 점수
  • 모델 라이선스/비용/크기 분석
  • 논문-코드-데모 연결
  • 이번 주 써볼 오픈소스
  • 과장/홍보성 표시
  • 보안 리스크 표시
Phase 5

커뮤니티와 지식베이스

한국 AI 실무자가 참고하는 지식 허브로 확장한다.

  • 이슈별 토론
  • 써본 후기/실험 로그
  • 좋은 댓글의 해설 반영
  • 용어사전과 인라인 설명
  • 자연어 질의 검색
  • 개념 관계도
Phase 6

프로덕트화

개인 프로젝트를 반복 사용되는 서비스로 만든다.

  • 공개 뉴스레터
  • 팀용 Watchlist
  • API 제공
  • 유료 전문가 리포트
  • 기업/제품/기술 타임라인
  • 국내 AI 생태계 전용 섹션

마이크로 UX 규칙

설명을 많이 붙여서 이해시키지 않는다. 같은 위치의 짧은 라벨, 상태 배지, 즉시 가능한 행동으로 제품 사용법이 드러나야 한다.

  • `이 기능은...` 대신 `알림 받기`, `비교 보기`, `원문 열기`, `실사용 점수`처럼 행동형 라벨을 쓴다.
  • 긴 태그 설명 대신 `공식`, `급등`, `검증 필요`, `국내 영향`, `논쟁 중` 상태 배지를 반복 사용한다.
  • 카드 본문은 최대 2줄이다. 더 필요한 내용은 `왜 중요` 확장 또는 상세 페이지로 넘긴다.
  • 숫자는 해석과 함께 표시한다. `+42%`가 아니라 `24시간 급등 +42%`로 쓴다.
  • 검색 결과는 `관련 결과 128개`보다 `가장 관련 높은 이슈 4개`를 먼저 보여준다.
  • 상세 페이지는 홈 요약을 반복하지 않는다. 요약 다음은 근거, 비교, 행동으로 이어진다.
  • 빈 상태는 긴 설명 대신 다음 행동 하나만 제공한다. 예: `관심 키워드 추가`.
  • 브리핑은 문단보다 블록이다. 모든 항목은 `결론`, `이유`, `행동`, `원문` 순서를 유지한다.
